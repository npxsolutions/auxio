## 5. Audit trail — ✅ GREEN (admin) / 🟡 YELLOW (user-side coverage)

- `app/admin/_lib/audit.ts` — `logAdmin()` writes `audit_log` rows with actor_id, actor_email, action, resource, resource_id, before/after JSON snapshots
- `app/api/admin/_lib/update.ts` — admin mutations trip the logger
- `audit_log` table included in the DSAR dump (§4)

**Gap:** User-initiated mutations (listing edits, channel connects, feed-rule changes) don't appear to trip the audit logger. PE/SOC 2 auditors want a single audit trail covering *all* privileged writes, not just admin console.

**Action:** Extend `logAdmin` → generic `logAudit`, call from every listing / channel / feed-rule mutation endpoint. ~1 day.

## 7. Metrics infrastructure — ✅ GREEN

`app/admin/metrics/page.tsx` computes live:
- MRR + ARR (via `PLAN_MRR × active subscribers`)
- Active / past-due / churned customer counts
- Plan distribution
- 6-month user-growth trend
- Active listings, total GMV, total profit
- Channel-type distribution
- **Rule-of-40 estimate** (growth rate computed from month-over-month new-user delta)

This is further than most pre-revenue SaaS get before Series A. Missing: NRR, GRR, logo churn, CAC payback, LTV/CAC — these need at least 3 months of paid customer data, so deferring is correct.

**Action:** Add the missing retention metrics to `/admin/metrics` at 30-day mark after first 10 founding partners are live. Template the new columns now (empty) so the dashboard has slots ready.
